PRODUCTO

Esta opción almacena toda la información básica de un producto y ofrece los siguientes campos:

  • Grupo: Almacena el grupo de producto al que pertenece.
  • Código: Almacena el código del producto.
  • Nombre: Almacena el nombre ó la descripción del producto.
  • Numero de Unidades: Almacena el numero de unidades que tiene el producto, como por ejemplo media botella de aguardiente posee doce tragos aproximadamente, a continuación se mostrará algunos ejemplos del funcionamiento de este campo:
    1. Si el valor almacenado es uno (1), se descontará una unidad a la existencia al momento de servir el pedido o al momento de cancelar la cuenta.
    2. Si el valor almacenado es cero (0), indicará que el producto tiene adiciones.
    3. Si el valor almacenado es diferente a uno (1) y a cero (0), indicará que el producto es un subproducto, y depende de un producto padre, po ejemplo una botella posee aproximadamente 23 tragos para ser gastado de una unidad de la botella; por ende al momento de vender un trago, automaticamente la existencia se mostrará en negativo hasta que llegue al valor del numero de unidades, para descargar la existencia del producto padre.
  • Producto Padre: Almacena el codigo del producto padre que tiene el producto, como por ejemplo un trago de aguardiente tiene como padre media botella o la botella completa.
  • Valor de Compra: Almacena el valor como se compro la unidad del producto.
  • Valor de Venta: Almacena el valor de venta del producto.
  • Stock Mínimo: Almacena el valor mínimo de existencias que debe tener el producto.
  • Stock Máximo: Almacena el valor máximo de existencias que debe tener el producto.
  • Existencia: Almacena la existencia actual del producto.
  • Activar alarma de stock: Almacena si el producto tendrá activo la alarma de stock durante la ejecución del programa.
  • Incluir este producto en las estadisticas: Almacena si el producto es parte de las estadisticas de inventario, para realizar el comparativo al momento de abrir la caja y al momento del cierre de la caja.
  • Limitar la existencia cero al momento de realizar la venta: Si está seleccionado, evalua si hay existencia para realizar la venta, sino hay existencia mostrará un mensaje que indicará que debe abastecer el inventario del producto a traves de la opcion de compras.
  • El producto exige preparación: Activa la acción de generar la impresión para efectos de preparación de algún producto.
